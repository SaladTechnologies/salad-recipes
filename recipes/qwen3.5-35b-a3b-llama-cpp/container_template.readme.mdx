# Qwen3.5-35B-A3B (Unsloth GGUF) with llama.cpp

This deployment runs `Qwen3.5-35B-A3B` with the official `llama.cpp` CUDA server and downloads the public GGUF weights automatically on startup.

<Callout variation="note">This recipe is public by default. If you want private access later, enable Container Gateway Authentication in the container group settings and include `Salad-Api-Key` in your requests.</Callout>

## Access

- <Link url={`https://${props.networking.dns}`}>Open the built-in llama.cpp UI</Link>
- `GET https://${props.networking.dns}/health`
- `GET https://${props.networking.dns}/v1/models`

## Defaults

- Model source: `unsloth/Qwen3.5-35B-A3B-GGUF:UD-Q4_K_XL`
- Model alias: `qwen3.5-35b-a3b`
- Startup profile: thinking enabled by default
- Sampling defaults: `temperature 0.6`, `top_p 0.95`, `min_p 0.0`, `top_k 20`

`temperature`, `top_p`, and `min_p` are startup defaults. They can still be overridden per request in your inference payload.

To control reasoning per request, add `/no_think` to disable thinking for that turn or `/think` to explicitly enable it.

## Curl Example

<CodeBlock language="bash">{`curl https://${props.networking.dns}/v1/chat/completions \\
    -X POST \\
    -H 'Content-Type: application/json' \\
    -d '{

If you enable auth later, add `-H 'Salad-Api-Key: ${props.apiKey}'` to your requests.
